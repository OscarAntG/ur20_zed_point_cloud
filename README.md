# ur20_zed_point_cloud

Point cloud processing ROS2 package for a Dry Waste Retrieval project. This repository integrates a ZED X stereo depth camera and a UR20 robotic arm using ROS2.
All point cloud processing is implemented in C++ and utilizes the Point Cloud Library (PCL).

## Dependencies
ROS2 (C++ `rclcpp`)
Point Cloud Library (PCL)

## `mask_point_cloud.cpp` (Primary Node)
The core ROS2 node for the Dry Waste Retrieval project. Subscribes to a ROS2 `Image` topic and retrieves a binary image mask generated by a SAM3-based image segmentation model (based on waste texture, color, and edges). 
Aligns the 2D mask to the live 3D point cloud data and highlights the corresponding pixels in red. Publishes both the isolated mask cloud and the highlighted point cloud.

## `filter_point_cloud.cpp`
Application-focused ROS2 node used to test point cloud segmentation techniques with PCL.

**Implemented features:**
* Passthrough filter
* Outlier removal
* Voxel downsampling
* SAC segmentation
* Cluster extraction
* Region growing segmentation
* LCCP segmentation

## `pcl_demo.cpp`
Testing node to become familiar with PCL and point cloud segmentation techniques.

**Implemented features:**
* Passthrough filter
* Outlier removal
* Voxel downsampling
* MLS polynomial estimation
* SAC segmentation
* Concave hull generation
* Region growing segmentation

## `echo_point_cloud.cpp`
ROS2 node that subscribes to a point cloud topic and publishes the received data. This node was used to test PCL implementation with ROS2 and to become familiar with the ROS2 `PointCloud2` interface and parameters.
The node also performs light processing such as voxel downsampling and RANSAC segmentation.

## `stats_point_cloud.cpp`
ROS2 node written as an exercise to determine the extent of depth camera noise with a statistics approach. It calculates the mean and standard deviation of the cloud and visualizes them as a 3D heatmap where regions of high deviation are mapped to red.
*(Note: Visualization is not fully functional).*
